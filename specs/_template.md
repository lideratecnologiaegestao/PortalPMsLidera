# Spec — <Módulo>

> Contrato que os subagents implementam. Mantida pelo `tech-writer` junto ao código.

## 1. Objetivo
O problema que o módulo resolve e para quem.

## 2. Conformidade legal
Leis/normas aplicáveis e como o módulo as atende.

## 3. Requisitos funcionais
Lista numerada, testável.

## 4. Requisitos não-funcionais
Desempenho, acessibilidade, segurança, privacidade específicos do módulo.

## 5. Modelo de dados
Tabelas, RLS, índices, relações. Aponta para `db/` quando já existe.

## 6. Contrato de API
Endpoints (método, rota, auth/role, request, response, erros).

## 7. Fluxos
Estados/sequência (Mermaid quando útil).

## 8. Integrações
Filas, IA, n8n, serviços externos.

## 9. LGPD/GDPR
Dados pessoais tratados, base legal, finalidade, retenção.

## 10. Critérios de aceite
Checklist objetivo (inclui teste de RLS quando há tabela nova).

## 11. Fora de escopo
O que esta entrega não cobre.
